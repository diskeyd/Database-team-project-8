# FMDS — Finance Management Database System

> 조선대학교 2026-1학기 데이터베이스 팀프로젝트
> 회사 부서별 수입·지출 관리 DBMS

## 프로젝트 개요

다중 사용자 환경에서 부서별 공금/예산을 안전하고 체계적으로 처리할 수 있는 DBMS 기반 시스템을 설계한다.

- **타겟**: 회사 부서별 재정 관리 (회사 → 부서 → 직원/담당자)
- **핵심 기술 포인트**: DBMS 동시 공유(Concurrent Sharing), 트랜잭션 무결성, 역할 기반 접근 제어
- **목표**: 기업/조직 재정 운영의 투명성과 효율성 증대

## 팀 구성

- 손영준
- 김기선
- 김동인

## 핵심 요구사항

1. **회계 담당자**: 수입·지출 거래 등록 (일자, 금액, 카테고리, 담당자 ID, 결제수단 필수 기록)
2. **감사 담당자**: 조건별(기간, 담당자, 카테고리) 거래 내역 조회 권한 — 수정 권한 제한
3. **관리자 (부서장)**: 예산 목표 대비 집행률 확인 및 리포트 조회
4. **시스템**: 동시 접근 시 데이터 정합성 보장 (트랜잭션) + 역할별 권한 제한

## 데이터 모델

6개 테이블로 구성:

| 테이블 | 역할 |
|--------|------|
| 사용자 | 시스템 사용자 계정 |
| 계좌 | 사용자별 계좌 (수입/지출 주체) |
| 거래내역 | 계좌 기준 수입·지출 기록 |
| 카테고리 | 거래 분류 (수입/지출 구분) |
| 목표 | 사용자별 저축 목표 |
| 예산 | 카테고리별 예산 설정 |
